## 1. Overview

This project implements a procedural character animation controller as a C++ DLL plugin for the N8RO simulation platform. The plugin drives a humanoid character through four motion states — Walk, Run, Hands Up, and Cheer and Wave — using mathematically generated joint angles computed in real time. No pre-recorded animation clips or motion capture data are used. All motion is produced through sine-wave functions, a state machine, and exponential joint interpolation evaluated every frame.

---

## 2. Objectives

- Generate all character motion procedurally through code
- Implement a state machine managing four distinct animation states
- Control 10 major joints of the humanoid skeleton per frame
- Respond to real-time keyboard input with immediate state transitions
- Interpolate smoothly between poses to eliminate visible snapping

---

## 3. System Architecture

The plugin is structured around four subsystems that execute sequentially each frame.

**Input Handler** reads keyboard state using `GetAsyncKeyState` and maps keys 1–4 to the four animation states. State changes are recorded with a timestamp to support time-based and gesture animations.

**State Machine** holds the current active state as an enum value (`WALK`, `RUN`, `HANDS_UP`, `CHEER`). Only one state is active at any time and transitions occur immediately on key press.

**Animation Generator** computes a target angle triplet `(x, y, z)` in radians for each of the 10 controlled joints based on the active state and simulation time.

**Interpolation and Output** blends current joint angles toward their targets using exponential smoothing and writes the results to the engine's joint override list each frame.

## 4. Motion States

### Walk
Sine-wave driven at 6.0 rad/s with leg swing amplitude of 0.40 rad and arm swing of 0.30 rad. Left and right hips are 180° out of phase. Arm swing is contra-lateral — left arm swings forward when right leg swings forward. Knee flexion scales proportionally with hip swing magnitude, producing a natural stride without an independent oscillator.

### Run
Identical structure to Walk with frequency doubled to 12.0 rad/s, leg amplitude increased to 0.70 rad, and arm amplitude to 0.60 rad. Elbow bend is introduced and scales with arm swing velocity, producing the pumping elbow motion characteristic of vigorous running. The result is visually distinct from Walk and clearly communicates a higher-energy movement.

### Hands Up
Both shoulders rotate upward symmetrically, with elbows bending naturally to complement the raised arm position. The pose is held for a fixed duration before the state machine automatically returns the character to a neutral stance. This state demonstrates time-based gesture animation and automatic state recovery.

### Cheer and Wave
One arm rests on the hip while the other performs a continuous waving motion driven by a sine function applied to the shoulder rotation. A slight weight shift is applied to the hip joints to give the pose a more natural, relaxed appearance. The waving motion repeats procedurally for as long as the state remains active, demonstrating how sine-wave functions can drive expressive repeating gestures.

## 6. Animation Mathematics

Locomotion is generated by the formula:

```
joint_angle = sin(t × frequency) × amplitude
```

Where `t` is simulation time in seconds, `frequency` controls cycle speed, and `amplitude` controls range of motion.

Bilateral symmetry is achieved by negating the value for the opposite side. Knee flexion is derived from the hip value:

```
knee_angle = abs(hip_swing) × scale
```

The Cheer and Wave state uses the same sine function to drive the waving arm:

```
wave_angle = sin(t × 8.0) × 0.5
```

Smooth transitions between all states use exponential smoothing each frame:

```
alpha   = 1 − exp(−15 × dt)
current += (target − current) × alpha
```

This converges quickly when the difference is large and slows as values approach their target, producing a natural ease-out without explicit animation curves.

## 7. Challenges and Solutions

**Joint axis identification** — The X, Y, and Z axes of each joint do not align with the global coordinate frame. Correct axes were identified by applying isolated test rotations and observing the resulting limb direction in the N8RO viewer. Each joint was calibrated independently before being incorporated into the animation functions.

**Animation tuning** — Small changes to frequency, amplitude, or axis direction produced significant visual differences. Parameters were adjusted incrementally through repeated build-and-test cycles until each state produced a natural and stable result.

**API joint limitations** — The plugin API exposes only the 10 limb joints listed above. Root translation and spine rotation are not available. Gesture states such as Hands Up and Cheer were designed entirely within these constraints, using shoulder and elbow positioning to communicate the intended pose clearly without torso or spine control.

**Transition stability** — When the engine evaluates frames with the same timestamp, dt becomes zero and smoothing stalls. This was resolved by clamping dt to a minimum of 16 ms when it falls outside a valid range, ensuring interpolation always progresses toward target values.

## 8. Results

The plugin loads correctly into N8RO and drives the character through all four motion states in response to keyboard input. Walk and Run produce visually convincing bipedal gait with correct leg alternation and contra-lateral arm swing. The Hands Up state raises both arms naturally and returns to neutral automatically. The Cheer and Wave state produces a stable, expressive repeating gesture with natural weight distribution. All state transitions are smooth with no visible discontinuities. Performance is well within real-time requirements as all computation is limited to floating-point arithmetic and map lookups per frame.

## 9. Conclusion

This project demonstrates that procedural animation using mathematical joint control, a state machine, and exponential interpolation is sufficient to produce convincing and varied humanoid motion in real time without any animation asset dependencies.
